How to Host a Free Website on GitHub Pages

A compact, copy-ready guide you can put on a handout or upload directly as an HTML page.

1. Create a GitHub Account

Sign up at github.com if you don’t already have an account. It’s free.

2. Create a New Repository

  1. Click New repository.
  2. Name it exactly username.github.io (replace username with your GitHub username).
  3. Set visibility to Public and initialize with a README.

3. Add Your Website Files

Upload your site files (HTML, CSS, JS). Two common ways:

  • Web interface: Drag-and-drop files into the repo.
  • Git (recommended for updates): Use the commands below from your project folder.
# initialize a repo (if you haven't already)
git init
git add .
git commit -m "Initial commit"

# add remote (replace USERNAME)
git remote add origin https://github.com/USERNAME/USERNAME.github.io.git

git branch -M main
git push -u origin main

Make sure your homepage file is named index.html.

4. Enable GitHub Pages

  1. Open the repo on GitHub and go to Settings → Pages.
  2. Under Source, pick the branch (usually main) and the root folder.
  3. Save. GitHub will publish your site at https://username.github.io.

It may take a minute or two for the site to become live. If you see a 404 right away, wait a bit and refresh.

5. (Optional) Connect a Custom Domain

If you own a domain (e.g., mywebsite.com), you can point it to GitHub Pages:

  1. Buy a domain from a registrar (Namecheap, GoDaddy, etc.).
  2. Create a file named CNAME in your repo with your domain name as its only content, e.g. mywebsite.com.
  3. At your registrar, add DNS records (A or ALIAS/ANAME or CNAME depending on setup) to point to GitHub’s servers. See GitHub Pages docs for exact records.
# CNAME file content (one line):
mywebsite.com

GitHub Pages supports HTTPS for custom domains — GitHub will provision a certificate after DNS is correctly set up.

Key Points

  • Free option: your site lives at username.github.io.
  • Custom domain: requires buying a domain; GitHub Pages supports it.
  • Best use cases: portfolios, docs, static sites, blogs (with Jekyll).
  • Limitations: GitHub Pages only hosts static sites — no server-side languages (PHP, databases).

Example

If your GitHub username is jerwin123 and your repo is jerwin123.github.io, your site will be available at:

https://jerwin123.github.io
